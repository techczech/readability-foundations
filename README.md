# Foundations of Readable and Accessible Documents

Evidence-based guidance for creating readable and accessible documents, based on cognitive load theory and reading strategy research.

**Author:** Dominik Lukeš, CC BY

## Overview

This repository contains the foundational content for the Readability Coach - guidance on making documents and presentations more readable and accessible.

The full guidance document is available at: [Readable and Accessible Templates](https://bit.ly/ox-templates)

## Five Principles of Readability

1. **Space** - More space around text = more attention for content
2. **Chunks** - Smaller chunks are easier to process, skim, and scan
3. **Guides** - Visual guides (bold, icons, summaries) help processing
4. **Information Structure** - Important information first, background last
5. **Simple Language** - Shorter sentences, address the reader, use verbs over nouns

## Key Recommendations

### For Documents
- Use headings marked with styles (not just bold)
- Break lists into bullet points
- Keep paragraphs short
- Bold key phrases (not entire sentences)
- Put important information first, background last
- Average sentence length: 15-20 words
- Line length: 50-70 characters

### For PowerPoints
- Minimum 28pt font for Arial
- Lines under 30 characters
- Use SmartArt instead of plain bullets
- No text over images
- Logos only on title and final slides
- Use animations sparingly to draw attention

## Reading Strategies

Documents should support three reading modes:
1. **Comprehensive** - reading start to finish
2. **Skimming** - reading for gist
3. **Scanning** - finding specific information

## Cognitive Load

Reduce cognitive load by:
- Creating shorter, meaningful chunks
- Providing visual guides to structure
- Reducing distractions and irrelevant content
